# GRouter Control Panel (Панель управления роутером)

Красивый и современный веб-интерфейс для мониторинга ресурсов, управления сервисами и автоматического обновления программ (например, `tg-ws-proxy` и `usque`) на роутере Xiaomi (Entware MIPS/MIPSel).

Панель состоит из двух частей:
1. **Веб-интерфейс (Frontend):** Статический сайт на HTML, CSS и Vanilla JS, который можно захостить на GitHub Pages или запускать локально.
2. **API-сервер (Backend):** Легковесный скрипт на Python 3 (`router_api.py`), работающий на самом роутере и выполняющий системные команды.

---

## Установка API-сервера на роутер

Для работы веб-панели вам нужно скопировать скрипты на роутер и настроить автозапуск.

### 1. Копирование файлов на роутер
Так как на роутере используется SSH Dropbear (порт `222`), вы можете передать файлы по SFTP/SCP. Если SFTP не настроен, используйте стандартные команды `scp` в PowerShell/Terminal на компьютере:

```powershell
# Копируем API-сервер в /opt/scripts/
scp -P 222 router_api.py root@192.168.1.1:/opt/scripts/router_api.py

# Копируем скрипт автозапуска в /opt/etc/init.d/
scp -P 222 S90router-api root@192.168.1.1:/opt/etc/init.d/S90router-api
```

### 2. Настройка прав и запуск
Подключитесь к роутеру по SSH (`ssh -p 222 root@192.168.1.1`) и выполните:

```sh
# Сделать файлы исполняемыми
chmod 755 /opt/scripts/router_api.py
chmod 755 /opt/etc/init.d/S90router-api

# Запустить API-сервер прямо сейчас
/opt/etc/init.d/S90router-api start
```

Скрипт `S90router-api` автоматически пропишется в автозапуск Entware и будет запускаться при каждой загрузке роутера. Логи работы пишутся в `/opt/var/log/router_api.log`.

## Публикация на GitHub Pages

Вы можете опубликовать веб-интерфейс на GitHub Pages:
1. Создайте новый публичный или приватный репозиторий на GitHub.
2. Инициализируйте гит и отправьте файлы `index.html`, `style.css` и `app.js`:
   ```bash
   git init
   git add index.html style.css app.js
   git commit -m "Initial commit"
   git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
   git branch -M main
   git push -u origin main
   ```
3. В настройках репозитория на GitHub перейдите в **Settings** -> **Pages**.
4. В разделе **Build and deployment** выберите источник **Deploy from a branch**, выберите ветку `main` (папка `/root`) и нажмите **Save**.
5. Через пару минут ваш сайт будет доступен по адресу `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПОЗИТОРИЯ/`.

## Важное при запуске из GitHub Pages (Mixed Content)

Поскольку GitHub Pages работает по безопасному протоколу `HTTPS`, а ваш роутер в локальной сети работает по `HTTP` (`http://192.168.1.1:8089`), браузер по умолчанию заблокирует запросы к роутеру из соображений безопасности (ошибка **Mixed Content** / Смешанный контент).

Чтобы это исправить, выберите один из вариантов:

### Вариант 1: Разрешить небезопасный контент для сайта (Рекомендуется)
Это абсолютно безопасно, так как запросы отправляются только на ваш собственный роутер внутри домашней сети:
1. Откройте ваш опубликованный сайт на GitHub Pages.
2. В адресной строке браузера слева от URL-адреса нажмите на значок настроек (обычно это **замочек** или **ползунки** <i class="fa-solid fa-sliders"></i>).
3. Нажмите кнопку **«Настройки сайтов»** (Site Settings).
4. Прокрутите список вниз, найдите параметр **«Небезопасный контент»** (Insecure content).
5. Измените значение с *Блокировать (по умолчанию)* на **«Разрешить»** (Allow).
6. Вернитесь на страницу панели управления и обновите её (F5).

### Вариант 2: Запуск локально
Локальные файлы не блокируются браузером. Вы можете просто скачать файлы проекта себе на компьютер и дважды кликнуть по файлу `index.html`, чтобы открыть его как `file:///C:/.../index.html`. Панель будет работать и подключаться к роутеру без дополнительных настроек браузера.

## Локальное тестирование на компьютере (без роутера)

Чтобы проверить работу интерфейса прямо на компьютере, в репозиторий добавлен имитатор API роутера:

1. Запустите имитатор на компьютере:
   ```bash
   python mock_router_api.py
   ```
   Сервер запустится локально на порту `8089`.
2. Откройте файл `index.html` в браузере.
3. В поле **Хост API** введите `http://localhost:8089` (или оставьте `http://192.168.1.1:8089`, если имитируете другую сеть).
4. Нажмите **Подключить**.
5. Вы сможете включать/выключать имитированные сервисы, наблюдать за графиком памяти и процессора, а также симулировать установку обновлений.
